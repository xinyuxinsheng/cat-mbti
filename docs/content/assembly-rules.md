# assembly-rules.md — 前端动态拼装规则

> 目标：用「分层内容库 + 触发规则」覆盖 模块A 136 种两猫组合、模块B 256 种人猫组合，
> **不逐组合写全文**。前端拿到两个 4 字母 code，按下列规则从 JSON 选块、排序、去重、拼装。
> 达标红线：模块A 解锁后 ≥6 块 / ≥500 字；模块B ≥7 块 / ≥600 字。文末 3 个实测示例证明任意组合达标。

---

## 0. 字母与字段约定

- 猫的 4 字母位：`[0]=E/I 社交性`、`[1]=S/N 好奇心`、`[2]=T/F 支配性`、`[3]=J/P 规律性`。
- 模块A：两只猫记为 `A`、`B`（谁是A谁是B不影响结果，规则对称）。
- 模块B：人记为 `human`，猫记为 `cat`。
- `when` 触发语法：`&&`=与、`||`=或、`!=`=两位字母不同、`always`=恒真。
  例：`A.T&&B.T` 两猫都强势；`A.T!=B.T` 一强一弱；`human.F&&cat.T` 人温和且猫强势。
- `ref` 指向 `relation-science.md` 的参考文献 `[1]-[11]`；`〔经验共识〕` 表示无可核验来源、按通识表述，前端照常展示，不必显示编号。
- **科学注脚渲染**：`science` 字段用小字/浅色，与正文的档案局幽默腔视觉分离（正文=解密档案的拟人吐槽，注脚=克制严谨）。

## 1. 免费层 vs 付费层（两模块通用）

- **免费可见**：缘分值/适配分 + 等级/称号 + 一句锐评（沿用现有 `pairing.js` 分桶逻辑，本任务不改）。
- **付费解锁**：下面第 2、3 节拼装出的全部内容块。免费层只给"钩子"，付费层给"厚度"。

---

## 2. 模块A（两猫对比）拼装算法

输入：`codeA`、`codeB`（各4字母）。按顺序产出块：

### 2.1 维度对比（固定 4 块，必出）
对 `EI / SN / TF / JP` 四个维度各选 1 块：
```
for dim in [EI, SN, TF, JP]:
    la = codeA[dim.idx]; lb = codeB[dim.idx]
    key = (la == lb) ? (la+la) : "MIX"     # 例: 都E→"EE"; 一E一I→"MIX"
    push dims[dim][key]
```
- 键规则：同字母取 `字母×2`（`EE/II/SS/NN/TT/FF/JJ/PP`），异字母取 `MIX`。
- 排序：固定 `EI → SN → TF → JP`（先社交、再好奇、后支配、末规律）。

### 2.2 冲突预警（选 3–4 块）
```
hits = [c for c in conflict if eval(c.when, A=codeA, B=codeB)]
specific = hits where when != "always"        # 具体触发的
# 按"命中字母数"降序排（越具体越靠前）：把 when 里的原子数当权重
specific.sort(key = count_atoms(when), desc)
pick = specific[:3]
if any hit with when=="always": pick += [该 always 块]   # 如 c16 如厕即信号 作为收尾
```
- 目的：优先展示"针对这对猫的具体冲突"，`always` 类（如厕信号）永远作为压轴的通用预警。
- 去重：同一 `title` 只保留一条（当前库无重复 title，规则留作扩展保险）。

### 2.3 和平共处（选 3–4 块）
```
hits = [p for p in peace if eval(p.when, A, B)]
conditional = hits where when != "always"     # 条件命中优先
picked = conditional[:2] + (always 命中里挑最相关的 2 条)
```
- 建议 always 池里固定优先 `p1 资源N+1铁律`、`p10 引入慢半拍`（可操作性最强），其余按需补。

### 2.4 输出结构
```
[header: 缘分值+称号+锐评]  ← 免费
[dims × 4]  [conflict × 3~4]  [peace × 3~4]   ← 付费
```
块数 ≥ 4+3+3+1 = 11，稳超 6 块红线。

## 3. 模块B（人猫适配）拼装算法

输入：`human`（人MBTI 4字母）、`cat`（猫MBTI 4字母）。

### 3.1 维度关系解读（固定 4 块，必出）
```
for dim in [EI, SN, TF, JP]:
    key = human[dim.idx] + "-" + cat[dim.idx]   # 人字母在前, 例 "T-F"
    push dims[dim][key]
```
- 每维 4 种组合（人×猫），共 16 键，全部已写透、无换皮。排序 `EI→SN→TF→JP`。

### 3.2 谁在驯服谁 · 权力结构（选 1–2 块）
```
hits = [p for p in power if eval(p.when, human, cat)]
# 主块 = 由 TF 维度决定的那条（human.T/F × cat.T/F 组合），最能定义权力关系
primary = hits 中 when 含 T/F 原子的那条
secondary = hits 中其余最具体的 1 条(可选)
push primary; if secondary: push secondary
```
- `power` 覆盖了 4 种 TF 基本盘（F×T 被拿捏 / T×F 温柔驯化 / T×T 宫斗 / F×F 共治）+ 6 种复合局，任意人猫至少命中 1 条 TF 基本盘，保证 primary 必有。

### 3.3 雷区清单（1 块，聚合猫 4 字母各 1–2 条）
```
mineTips = []
for X in cat 的 4 个字母:              # 如 ISFP → [I,S,F,P]
    mineTips += mines[X][:2]           # 每字母取前2条
render as 一个"雷区清单"块（bullet 列表, 4~8 条）
```

### 3.4 讨好攻略（1 块，聚合猫 4 字母各 1–2 条）
```
pleaseTips = []
for X in cat 的 4 个字母:
    pleaseTips += please[X][:2]
render as 一个"讨好攻略"块（bullet 列表, 4~8 条）
```

### 3.5 一周名场面（可选加分，选 1–2 条）
```
hits = [w for w in weekly if eval(w.when, human, cat)]
push hits[:2]   # 命中就放，是幽默彩蛋，锦上添花
```

### 3.6 输出结构
```
[header: 适配分+等级]  ← 免费
[dims × 4]  [power 主(+补)]  [雷区清单 ×1]  [讨好攻略 ×1]  [weekly ×0~2]   ← 付费
```
块数 ≥ 4+1+1+1+1 = 8，稳超 7 块红线。

## 4. 边界与合规（前端渲染须遵守）
- `ref` 为 `〔经验共识〕` 的块：正常展示，不显示文献编号，不得伪造成"研究表明"。
- 任何含"乱尿/拒食/持续躲藏/突发攻击"字样的块，尾部保留"持续出现请咨询兽医"，不可裁掉。
- 涉及数字（依恋约2/3、社交偏好约半数）的注脚用"约/研究估计"措辞，勿改成精确小数。
- 主人人格 ↔ 猫健康只说"相关"，前端文案禁止改写成因果宣称。

## 5. 三个完整拼装示例（均为实测字数，证明达标）

> 字数=去空白后的中文字符数；块数含免费 header 1 块。

### 示例① 模块A：ESTJ猫 × INFP猫
- 两 code：A=`ESTJ`[E,S,T,J]，B=`INFP`[I,N,F,P]
- **维度块（4）**：
  - EI：E≠I → `MIX` →「一动一静」
  - SN：S≠N → `MIX` →「一探一守」
  - TF：T≠F → `MIX` →「强弱格」
  - JP：J≠P → `MIX` →「一准一浪」
- **冲突命中 7 条**，按具体度取前 3 + always 收尾：
  `c12 强势规律vs温和随性`（4原子最具体）、`c2 试探性打压`、`c13 热情越界` + `c16 如厕即信号`(always压轴)
- **和平命中 13 条**，取条件 2 + always 2：
  `p6 强势方吃第一口`、`p7 给内向方保留区` + `p1 资源N+1铁律`、`p10 引入慢半拍`
- **产出**：13 块，**≈694 字** ✅（红线 6块/500字）
- 解读逻辑自洽：一只霸总规律猫(ESTJ) × 一只迷糊随性猫(INFP)，四维全错位→冲突聚焦"强势方压制+作息打架"，和平方案给弱势内向方留退路、按 N+1 分资源。

### 示例② 模块A · 镜像边界：ISTJ猫 × ISTJ猫（证明"同型/低差异"组合也够厚）
- A=B=`ISTJ`[I,S,T,J]
- **维度块（4）**：II「双隐士」、SS「双保守派」、TT「双王座」、JJ「双钟表」（全同字母，取 `字母×2`）
- **冲突命中 5 条**：`c15 保守双雄对峙`、`c1 王座之争`、`c8 沉默的冷战` + `c16 如厕即信号`
- **和平命中 11 条**：`p6 强势方吃第一口`、`p7 给内向方保留区` + `p1`、`p10`
- **产出**：13 块，**≈709 字** ✅
- 说明：即便两只猫一模一样（缘分算法给"镜像猫生"），内容库靠"双X"维度块 + 双强势/双内向专属冲突，仍拼出 700+ 字，不会因为"没差异"而单薄。

### 示例③ 模块B：INTJ人 × ISFP猫
- human=`INTJ`[I,N,T,J]，cat=`ISFP`[I,S,F,P]
- **维度块（4）**：
  - EI：I-I →「双静修」
  - SN：N-S →「你野它稳」
  - TF：T-F →「你管它顺」
  - JP：J-P →「你准它浪」
- **权力结构命中 2 条**：主块 `温柔驯化`(human.T&&cat.F，TF基本盘) + 补块 `秩序拉锯`(human.J&&cat.P)
- **雷区清单**：聚合猫字母 I,S,F,P 各 1 条 → 「强行抱逼社交 / 环境剧变应激 / 大声呵斥默默受伤 / 强上死板时间表逆反」
- **讨好攻略**：聚合 I,S,F,P 各 1 条 → 「让它来找你 / 一切慢慢来 / 回应它安静的示好 / 随它的节奏来」
- **一周名场面命中 3 条**，取 1：`周二(I-I) 全天零对话都觉得完美`
- **产出**：10 块，**≈734 字** ✅（红线 7块/600字）
- 爽点落在"人"身上：一个冷静掌控型主人(INTJ) × 一只谨慎温顺猫(ISFP)，权力判词是"你以为你在管它，其实是温柔驯化"，全程第二人称教你怎么拿捏。

## 6. 覆盖性论证（为什么不会漏组合）
- **维度块**：模块A 每维 3 键(同同/异)恰好覆盖任意两字母关系；模块B 每维 4 键覆盖人×猫全部组合。4 维必出 → 任意组合都有 4 块打底。
- **冲突/和平/power**：`when` 规则用字母布尔表达，任意 code 代入必有 `always` 兜底（模块A的 c16/p1/p3/p5/p9/p10/p13 恒真；模块B power 的 4 条 TF 基本盘互斥且穷尽 human.T/F × cat.T/F 四象限，必命中 1 条）。故**不存在拼不出内容的组合**。
- **厚度**：4 维度块单块 100–120 字，仅维度块就 ≈450–480 字；叠加冲突/和平/权力/清单后稳超红线（实测最低样本亦 690+/730+ 字）。
